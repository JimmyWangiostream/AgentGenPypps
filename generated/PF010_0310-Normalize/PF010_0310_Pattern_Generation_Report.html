AgentGeneratePypps · revised generation method

Flow-scoped retrieval + stateful flow-level Pattern generation

這份新版報告說明修正後的流程：不是整份 TC 一次 gen，也不是每個 step 孤立 gen，而是保留 flow 內的 ordered steps、變數、狀態與 dependency，再產生一個 TC 對應的一份 Pattern。

為什麼要改

整份 TC 一次 gen 的問題

retrieval 太廣，SET/CLEAR、SSU/POR/LINKSTARTUP、Enable/Flush expected value 容易混在一起。

每個 step 孤立 gen 的問題

step 之間的 write_record、reset_type、previous value、expected value 會斷掉。

現在採用的中間解

Flow-scoped retrieval + stateful flow-level generation。每個 flow 整體生成，最後組成一個 Pattern。

新版 Pipeline

01Parse full TC

建立 global context、Flow IR、Step IR。

02Build dependency graph

抓 producer / consumer / shared state。

03Retrieve per flow

每個 flow 查 LLM Wiki 與 PyPPS code refs。

04Decide per flow

用 source priority + flow state 做 decision。

05Generate flow method

一個 flow 生成一個 stateful method。

06Assemble Pattern

一個 TC 仍然輸出一個 Pattern .py。

07Artifacts

retrieval / assumptions / validation 都 flow-scoped。

08Validate

py_compile、AST、flow coverage、dependency check。
